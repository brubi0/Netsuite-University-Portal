Custom KPIs from Saved Searches

When the 75+ standard KPIs don't meet your needs, you can create custom KPIs based on saved searches. This lets you track any metric you can query with a saved search ‚Äî from custom calculations to specific transaction types.

Requirements for Custom KPIs

‚ö†Ô∏è Critical Requirements

A saved search must meet these requirements to work as a KPI:

  • Exactly ONE summary field in Results (Sum, Count, Average, etc.)
  • Date as Available Filter ‚Äî NOT as Criteria filter
  • No date criteria ‚Äî the KPI portlet applies date ranges dynamically

Creating a Custom KPI Search

üìç Steps

  1. Create a saved search on the desired record type
  2. Add criteria filters (EXCEPT date filters)
  3. On the Results subtab, add ONE field with a Summary Type (Sum, Count, etc.)
  4. On the Available Filters subtab, add the Date field
  5. Save the search
  6. Navigate to Setup ‚Üí Company ‚Üí KPIs
  7. Add your custom KPI based on the saved search

üéØ Key Takeaways

  • Custom KPIs are based on saved searches
  • Must have exactly one summary field (Sum, Count, Average, Min, Max)
  • Date must be an Available Filter, not a Criteria filter
  • KPI portlets apply date ranges dynamically
  • Create KPIs at Setup ‚Üí Company ‚Üí KPIs
