What is Joined Data?

Joined data allows you to include fields from related records in your saved search results. For example, when searching transactions, you can include the customer's phone number, the sales rep's email, or the item's category ‚Äî all in the same search without creating multiple searches.

Understanding Joins

NetSuite uses ellipsis notation (...) to indicate that a field is from a related (joined) record type.

Sales Rep... Email

This notation means: "Go to the Sales Rep record that's linked to this transaction, and retrieve the Email field from that record."

Adding Joined Fields

  1. On the Results subtab, scroll through the field dropdown
  2. Look for fields with ellipsis (...) ‚Äî these are from related records
  3. Select the joined field (e.g., "Sales Rep...Email")
  4. Click Add
  5. The field is added to your results columns

When you join to a record that has multiple related records, your search may return duplicate rows. For example, if a customer has 5 transactions and you search Customers with joined Transaction data, you'll see the customer listed 5 times ‚Äî once for each transaction.

üí° Pro Tip: Custom Labels for Joined Fields

  • "Sales Rep...Email" ‚Üí "Rep Email"

Using Joins in Criteria

You can also use joined fields in the Criteria subtab to filter based on related record data.

üéØ Key Takeaways

  • Joined data pulls fields from related record types
  • Look for ellipsis notation (...) to identify joined fields
  • Use joins to avoid creating multiple searches for related data
  • Be aware of one-to-many relationships that may cause duplicate rows
  • Use Custom Labels to simplify joined field column headers
  • Joined fields can be used in both Results and Criteria
