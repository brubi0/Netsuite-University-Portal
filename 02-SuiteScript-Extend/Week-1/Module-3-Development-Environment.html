üìÑ File Templates

File templates provide boilerplate code for each script type, saving you from writing the same setup code repeatedly. They include proper JSDoc annotations, module imports, and entry point functions.

Why Use Templates?

  • Consistent code structure across all scripts
  • Correct @NApiVersion and @NScriptType tags
  • Pre-configured entry points (beforeLoad, pageInit, etc.)
  • Proper module dependency setup

BurkyBang Templates (Recommended)

Adam Smith (BurkyBang) maintains excellent SuiteScript templates for WebStorm:

  1. Visit github.com/BurkyBang
  2. Find the WebStorm File Templates repository
  3. Clone or download the SuiteScript 2.1 templates
  4. Follow the installation instructions in the README
  5. Restart your IDE to activate the templates

üìù Type Declarations

Type declarations teach your IDE about SuiteScript modules. Without them, your editor doesn't know that record.load() returns a Record object or what methods are available.

What Type Declarations Provide

Autocomplete

Type record. and see all available methods

Parameter Hints

See required and optional parameters as you type

Return Types

Know what each function returns

Error Detection

Catch typos and incorrect usage before uploading

Installation

  1. Download type declarations from BurkyBang's repository (SuiteScript 2.0 folder)
  2. Place the .d.ts files in your project or a shared location
  3. Configure your IDE to recognize the type definitions
  4. WebStorm: Settings ‚Üí Languages & Frameworks ‚Üí JavaScript ‚Üí Libraries

üé® IDE Features in Action

Syntax Highlighting

    fieldId: 'entity'  // Orange: Strings

Autocomplete

After typing form., your IDE shows all available methods. Press Tab to insert the selected option:

    id: 'custpage_myfield',
    type: serverWidget.FieldType.TEXT,
    label: 'My Custom Field'

Error Detection

Red underlines indicate syntax errors before you even upload to NetSuite. Hover to see the error message:
