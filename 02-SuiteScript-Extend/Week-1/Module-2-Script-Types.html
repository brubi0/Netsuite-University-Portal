SuiteScript Types

Understanding the five most common script types and their use cases

üîÑ Server-Side vs Client-Side

Before diving into script types, it's essential to understand where scripts execute:

üñ•Ô∏è

Server-Side

Runs on NetSuite's servers. Cannot interact with the browser. Has access to all N/ modules.

üåê

Client-Side

Runs in the user's browser. Can update the UI in real-time. Limited module access.

üí° JavaScript Foundation

SuiteScript is built on JavaScript. A solid understanding of JavaScript fundamentals (variables, functions, objects, arrays, async/await) is essential for this course.

üìã The Five Core Script Types

1. User Event Script Server-Side

Runs automatically when a record is created, loaded, edited, or deleted. Executes on NetSuite's servers before the record reaches the user's browser or after it's saved.

üíº Practical Example

Lock commission fields so sales managers cannot modify their own information. The restriction happens server-side before the form ever reaches the browser‚Äîthere's no way to bypass it.

2. Client Script Client-Side

Runs in the user's browser, providing real-time field validation, UI updates, and interactive experiences. Cannot access server-side operations directly.

üíº Practical Example

Display a running commission total that updates in real-time as sales managers add lines to sales orders. The user sees the amount change immediately without saving.

3. Suitelet Server-Side

Creates custom pages within NetSuite. Used for custom data entry forms, reporting interfaces, or any UI that doesn't fit standard NetSuite records.

üíº Practical Example

Build a custom data entry form styled exactly as the company requires, with specific workflow and validation rules, before submitting data to standard NetSuite records.

4. Map/Reduce Script Server-Side

Designed for processing large datasets in parallel. Breaks work into stages (Get Input ‚Üí Map ‚Üí Reduce ‚Üí Summarize) and handles automatic rescheduling if governance limits are reached.

üíº Practical Example

Add a prefix to all item names across thousands of items. A search retrieves all items, and the Map/Reduce processes each one individually, distributing work across multiple threads.

5. RESTlet Server-Side

Provides a RESTful API endpoint for external systems to interact with NetSuite. Receives HTTP requests (GET, POST, PUT, DELETE) and returns structured responses.

üíº Practical Example

Integrate Shopify with NetSuite. The RESTlet receives order data from Shopify, interprets it, and creates the corresponding Sales Order and Customer records in NetSuite.

üìä Script Type Comparison

Script Type Execution Trigger Primary Use
User Event Server Record action (create/edit/delete) Automation, validation, field defaults
Client Script Browser User interaction UI enhancement, real-time validation
Suitelet Server URL access Custom pages, forms, reports
Map/Reduce Server Schedule or manual trigger Batch processing, mass updates
RESTlet Server HTTP request External integrations, APIs

üîß Other Script Types

While these five are the most common, NetSuite offers additional script types for specialized scenarios:

Script Type Use Case
Scheduled Script Time-based execution (hourly, daily, weekly)
Workflow Action Script Custom logic within SuiteFlow workflows
Portlet Dashboard widgets and KPI displays
Mass Update Script Bulk record updates via the Mass Update feature
Bundle Installation Script Pre/post-installation logic for SuiteBundles

üìö Certification Note

The NetSuite SuiteScript Developer certification tests your understanding of when to use each script type. Focus on understanding the triggers, execution context, and typical use cases for each.

üìñ Finding This in the Docs

To explore script types in NetSuite Help:

  1. Go to docs.oracle.com and search "SuiteScript 2.x Script Types"
  2. Navigate to SuiteScript 2.x API Reference ‚Üí Script Types
  3. Click on any script type (e.g., "User Event Script Type") for entry points and context objects

Key pages to bookmark:

  • User Event Script Type ‚Üí beforeLoad, beforeSubmit, afterSubmit
  • Client Script Type ‚Üí pageInit, fieldChanged, saveRecord
  • Map/Reduce Script Type ‚Üí getInputData, map, reduce, summarize
  • Suitelet Script Type ‚Üí onRequest entry point

üéØ Key Takeaways

  • User Event scripts run server-side on record actions‚Äîideal for automation and enforced validation
  • Client Scripts run in the browser for real-time UI updates and user interaction
  • Suitelets create custom pages that don't fit standard record patterns
  • Map/Reduce scripts handle large-scale batch processing with parallel execution
  • RESTlets provide API endpoints for external system integrations
  • Choose script type based on: trigger, execution context, and business requirement
