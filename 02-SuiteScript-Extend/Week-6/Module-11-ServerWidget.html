N/ui/serverWidget Module

Building custom forms, fields, sublists, and interactive interfaces

üé® Creating Forms

The serverWidget module enables creation of custom pages and forms in Suitelets:

const serverWidget = require('N/ui/serverWidget');

function onRequest(context) {
    const form = serverWidget.createForm({
        title: 'Custom Data Entry'
    });
    
    // Add fields, sublists, buttons...
    
    context.response.writePage(form);
}

üìù Adding Fields

// Text field
form.addField({
    id: 'custpage_name',
    type: serverWidget.FieldType.TEXT,
    label: 'Customer Name'
});

// Select field with options
const statusField = form.addField({
    id: 'custpage_status',
    type: serverWidget.FieldType.SELECT,
    label: 'Status'
});
statusField.addSelectOption({ value: '', text: '' });
statusField.addSelectOption({ value: '1', text: 'Active' });
statusField.addSelectOption({ value: '2', text: 'Inactive' });

// Date field
form.addField({
    id: 'custpage_date',
    type: serverWidget.FieldType.DATE,
    label: 'Start Date'
});

üìä Adding Sublists

const itemSublist = form.addSublist({
    id: 'custpage_items',
    type: serverWidget.SublistType.INLINEEDITOR,
    label: 'Line Items'
});

itemSublist.addField({
    id: 'custpage_item',
    type: serverWidget.FieldType.SELECT,
    label: 'Item',
    source: 'item'  // Populate from item record
});

itemSublist.addField({
    id: 'custpage_qty',
    type: serverWidget.FieldType.INTEGER,
    label: 'Quantity'
});

Sublist Types

TypeDescription
EDITORFull edit mode, one line at a time
INLINEEDITOREdit directly in grid
LISTRead-only display
STATICLISTNon-interactive display

üîò Adding Buttons

// Submit button (built-in)
form.addSubmitButton({
    label: 'Save'
});

// Custom button (calls client script function)
form.addButton({
    id: 'custpage_process',
    label: 'Process Records',
    functionName: 'processRecords'
});

üîó Attaching Client Scripts

// Link a client script to the form
form.clientScriptModulePath = './my_cs_form.js';

‚úÖ Client Script Functions

Button functionName values call functions in the attached client script. The client script must export those functions.

üì• Handling Form Submission

function onRequest(context) {
    if (context.request.method === 'GET') {
        // Display the form
        const form = serverWidget.createForm({ title: 'Data Entry' });
        // ... add fields
        context.response.writePage(form);
    } else {
        // Process POST submission
        const name = context.request.parameters.custpage_name;
        const status = context.request.parameters.custpage_status;
        
        // Create record, send email, etc.
        record.create({
            type: 'customer',
            values: { companyname: name }
        }).save();
        
        // Redirect or show confirmation
    }
}

üìñ Finding This in the Docs

To look up serverWidget form building:

  1. Go to docs.oracle.com ‚Üí SuiteScript 2.x Modules
  2. Click on N/ui/serverWidget Module
  3. Review Form, Field, Sublist, and Button objects

Key pages to bookmark:

  • serverWidget.createForm(options) ‚Üí form initialization
  • Form.addField(options) ‚Üí id, type, label, source parameters
  • serverWidget.FieldType enum ‚Üí TEXT, SELECT, DATE, CHECKBOX, etc.
  • Form.addSublist(options) ‚Üí sublist types (EDITOR, INLINEEDITOR, LIST)
  • Form.addButton(options) ‚Üí custom action buttons
  • Form.clientScriptModulePath ‚Üí attaching client scripts

üéØ Key Takeaways

  • serverWidget.createForm() creates a custom form
  • Field IDs must start with custpage_
  • FieldType enum provides TEXT, SELECT, DATE, CHECKBOX, etc.
  • SublistType.INLINEEDITOR enables grid editing
  • Use clientScriptModulePath to attach client-side logic
  • Check context.request.method to handle GET vs POST
  • Access submitted values via context.request.parameters
