Week 2: Client-Side Processing

Building Dynamic, Interactive Forms

Week 2 Overview

This week focuses on client-side workflow processing—actions that execute in the user's browser as they interact with forms. You'll master client triggers and learn to build dynamic forms that respond intelligently to user input, validate data in real-time, and provide immediate feedback.

By mastering client triggers, you'll create intuitive user experiences that guide users through data entry, prevent errors before save, and make forms adapt dynamically to user choices.

Week 2 Modules

02

Understanding Client Triggers

Learn the difference between client and server triggers. Master form-level triggers (Before User Edit, Before User Submit) and field-level triggers (Before Field Edit, After Field Edit, After Field Sourcing) to create dynamic, responsive forms.

03

Configuring Actions on Data Entry

Configure common client-side actions: Set Field Display Type, Set Field Mandatory, Set Field Value, Return User Error, Show Message, and Confirm. Master the Visual Builder to create complex conditions that control when actions execute.
