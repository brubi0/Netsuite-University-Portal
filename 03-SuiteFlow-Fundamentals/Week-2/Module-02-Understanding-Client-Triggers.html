Understanding Workflow Triggers

Workflow triggers are the foundation of controlling when your workflows execute specific tasks. They represent specific events that occur during the processing of a record in NetSuite and dictate when actions should be performed or when records should transition between states.

üí° The Most Important Concept

Triggers OCCUR, but workflows, actions, and transitions EXECUTE. This distinction is fundamental to understanding workflow timing and behavior!

Client Triggers Overview

Client triggers respond to events occurring in the user's web browser. They fire just before a record is displayed, while the user is interacting with the form, and when saving a record.

‚ö†Ô∏è Important Limitation

Client triggers only occur when creating a new record or editing an existing record. They do NOT occur when viewing a record!

Form-Level Client Triggers

Trigger When It Occurs Common Uses
Before User Edit When a user creates/edits a record‚Äîbetween request and display in browser Show/hide fields, make fields mandatory/optional, default values
Before User Submit When the user makes changes and saves the record Form-level validations, display validation error messages

Field-Level Client Triggers (Dynamic Forms)

üîπ Before Field Edit

Occurs after the user tabs out of a field but before data moves to memory‚Äîregardless of whether the value changed. Used for field-level validation.

üîπ After Field Edit

Occurs only when leaving a field and the field's value was actually changed. Used to affect other fields based on current field value.

üîπ After Field Sourcing

Similar to After Field Edit, but waits for value sourcing to complete. Used when selecting from list/record fields where values are sourced from related records.

üåü Key Takeaways

  • Triggers OCCUR, workflows/actions/transitions EXECUTE‚Äîfundamental to understanding timing
  • Client triggers only fire when creating or editing records‚ÄîNOT when viewing
  • Use Before User Edit to default values and set display characteristics as the form loads
  • Use Before User Submit to validate form fields during save
  • Before/After Field Edit and After Field Sourcing create dynamic forms
