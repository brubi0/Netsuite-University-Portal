Formulas Overview

Formulas represent calculations performed on one or more variables. In SuiteFlow, formulas enable complex logic beyond what the Visual Builder provides.

Formula Use Cases

  • Calculate fields: New salary = current salary √ó percent increase
  • Concatenate fields: Account code = phone number + postal code
  • Extract portions: Get domain name from email address
  • Reformat fields: Display negatives in parentheses
  • Validate fields: Error if zip code < 5 digits

üìã Dynamic Calculation

Formula values are calculated dynamically when the workflow instance runs. They can be used to conditionally initiate workflows or execute actions and transitions.

Formula Types

Type Trigger Use Case
SQL Formulas Server triggers Database operations, server-side processing
SuiteScript Formulas Client triggers Browser-side processing (covered in Module 7)

‚úÖ Performance Tip

SQL formulas execute against the NetSuite database on the server‚Äîkeeping them close to the database improves performance.

Formula Construction Elements

1. Internal Field IDs

Reference fields using their internal ID enclosed in curly braces:

{custrecord_phone_number}
{customer.phone}

2. SQL Functions

Use SQL functions in conditions for workflow initiation or action/transition conditions on server triggers:

LENGTH({custrecord_rating_code}) < 10

3. JavaScript/SuiteScript Functions

Use for client trigger conditions (covered in Module 7):

nlapiGetFieldValue('custrecord_rating_code').length < 10

SQL Formula Syntax

Field References

{field_id} -- Base record field
{record.field_id} -- Related record field
{custrecord_custom_field} -- Custom field

Common SQL Functions

Function Purpose Example
LENGTH() Get string length LENGTH({phone}) > 10
UPPER() / LOWER() Change case UPPER({name})
SUBSTR() Extract substring SUBSTR({email}, 1, 5)
NVL() Handle nulls NVL({amount}, 0)
TO_CHAR() Format numbers/dates TO_CHAR({amount}, '999,999')
CASE WHEN Conditional logic CASE WHEN {status} = 'A' THEN 'Active' ELSE 'Inactive' END

Formula Examples

Calculate New Salary

{custrecord_current_salary} * (1 + {custrecord_increase_pct}/100)

Concatenate Fields

{phone} || '-' || {zipcode}

Extract Email Domain

SUBSTR({email}, INSTR({email}, '@') + 1)

Validate Field Length

LENGTH({custrecord_rating_code}) < 3

Increment Counter

{custrecord_counter} + 1

üí° Return User Error Example

Use formula in Return User Error action condition to validate that rating code is A, B, or C:

UPPER({custrecord_rating_code}) NOT IN ('A', 'B', 'C')

Formula Best Practices

  • Always enclose field IDs in curly braces { }
  • Use SQL formulas with server triggers only
  • Test formulas with various data scenarios
  • Consider null values‚Äîuse NVL() to handle them
  • Keep formulas as simple as possible for maintainability
  • Document complex formulas for future reference

‚ö†Ô∏è Trigger Mismatch Warning

If you configure a SQL formula on a client trigger, it will be ignored and show "ERROR: Invalid expression" in the workflow execution log. Always match formula type to trigger type.

üåü Key Takeaways

  • Formulas enable complex logic beyond Visual Builder
  • SQL formulas work with server triggers
  • SuiteScript formulas work with client triggers (Module 7)
  • Reference fields using curly braces: {field_id}
  • Use standard SQL functions: LENGTH, UPPER, SUBSTR, NVL, CASE
  • Formulas calculate dynamically at workflow runtime
  • Always match formula type to trigger type
